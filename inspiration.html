Designer & design-engineer portfolios

Confident typography, intentional whitespace, tasteful motion.

  • Emil Kowalskiemilkowal.ski

    Single-column, content-first — no hero, no nav chrome. Hover any project tile and you get a frame-perfect easing demo that doubles as a credential for his animation work.

  • Paco Courseypaco.me

    Reads like a plain text file that happens to be beautifully kerned. Short intro, a Building list, a Writing list, a Now section, separated only by thin rules — the restraint forces you to read prose.

  • Mariana Castilho — UI Labsuilabs.dev

    Vertical scroll of self-contained interactive component vignettes on a near-white canvas. Each demo ships with its tech badge inline — portfolio and live spec sheet without ever feeling like Storybook.

  • Brian Lovinbrianlovin.com

    Two-pane macOS-style layout: persistent left sidebar, right pane swaps without a full reload. Treats a personal site as a desktop app without skeuomorphic kitsch.

  • Jim Nielsenjim-nielsen.com

    Defiantly plain — high-contrast black on warm off-white, generous measure, metadata in small monospace. No motion. A study in how confident hierarchy and one good text size can carry an entire site.

Writers & essayists

Sites designed to be read. Beautiful body type, sensible measure, almost no chrome.

  • Tom MacWrightmacwright.com

    Inlined stylesheet, system sans, 640px column, line-height 1.6. Honest, native-feeling text with no webfont flicker. Reads like a well-kept logbook.

  • A Working Library — Mandy Brownaworkinglibrary.com

    Patient narrow column of serif body, italicised book titles and source attributions threading through every essay. Structured as a reading practice (Reading / Writing / Thinking), not a blog.

  • Craig Modcraigmod.com

    A book-designer's web page: humanist serif at a generous size, wide margins, dinkus dividers, figure captions in italic small-caps. Designed to be read on a tablet at arm's length.

  • James Somersjsomers.net

    Georgia, full stop. A long centred column, near-zero chrome (just a header and a chronological list), footnotes and inline links doing all the visual work. The archaic styling is the point.

  • Steph Angostephango.com

    Aggressively minimal monochrome, tag-based navigation instead of a feed. Short evergreen notes cross-linked like a wiki — embodies his stated principle: style is consistent constraint.

Typography-first

Sites where the type choices are the entire design system.

  • Matthew Butterick — Practical Typographypracticaltypography.com

    Set in Butterick's own MB Type families with a live font-switcher (Heliotrope, Valkyrie, Century Supra, Triplicate). Strict 66-character measure, real small caps, hanging punctuation. The canonical example of the rules applied.

  • Robin Rendlerobinrendle.com

    Custom-licensed MD IO and MD System UI from Mass-Driver, separated sans/title/mono tokens, and a notes/stories/newsletter split that mirrors how a writer actually thinks. Probably the most considered web type stack online.

  • Frank Chimerofrankchimero.com

    Restrained sans for nav against a warm serif for prose, one accent colour, obvious 8pt vertical rhythm. Hierarchy is almost entirely typographic — leading, indent, a tight type scale.

  • Tonsky — Nikita Prokopovtonsky.me

    Read-the-books type-nerd: tight measure, real fi/fl ligatures, hyphenation enabled, custom monospace (he ships Fira Code), unicode dinkus stars marking editor's picks. Functional, not decorative.

  • Jason Kottke — kottke.orgkottke.org

    A study in long-form link-blog typography. A single readable column, italic source attributions, hierarchy built only from leading and indent. Has looked broadly the same since 1998 because the type was right the first time.

Programmer blogs

Information-dense but elegant. Confident type, no fluff.

  • Amos Wenger — fasterthanli.mefasterthanli.me

    Dark theme with serif body and unusually careful syntax highlighting. Inline asides as conversational character cards are a signature — footnotes that live in the flow without breaking it.

  • Armin Ronacher — lucumrlucumr.pocoo.org

    Auto/light/dark mode done quietly. Single-column reading layout that trusts the type. Code blocks feel like part of the prose rather than embedded screenshots. No popups, no newsletter modal.

  • Alex Kladov — matkladmatklad.github.io

    The most disciplined just-text blog of the bunch. Generous line-height, narrow measure, tiny top nav (About / Links / Blogroll) and nothing else. Unboxed code so Rust/Zig reads as part of the argument.

  • Hillel Waynehillelwayne.com

    Confident serif heads, sans body, a horizontal nav that says everything in five words. No sidebar, no related-posts grid — you get the post and a link to the next thing.

  • Bartosz Ciechanowskiciechanow.ski

    Minimalist sans body with bespoke interactive diagrams inline. The visual is the explanation, the text gets out of the way.

Brutalist but elegant

Raw HTML aesthetic, system fonts or careful monospace, almost no styling — but the structure carries it.

  • Dan Luudanluu.com

    Black serif on white, no header image, no nav chrome, posts listed by date. The absence of design signals confidence in the prose — and the prose earns it.

  • Oskar Wickströmwickstrom.tech

    Monospace throughout, ch-based column widths, ASCII rules between sections. Reads like a well-kept terminal buffer. He authored "The Monospace Web" — that's the design statement.

  • Ben Kuhnbenkuhn.net

    Plain serif body, navigation as bracketed text links ([home] [say hi] [blogroll]), section headers as dashed underlines. A tidy plaintext README that happens to render in a browser.

  • Maciej Cegłowski — Idle Wordsidlewords.com

    Narrow single column, serif body, a small toothfish in the corner as the only ornament. Shaped like an essay magazine, not a CMS theme.

  • Ted Unangst — flaktedunangst.com

    Default-ish serif, flat post list, no sidebar, no taxonomy, no avatar, no related-posts. The discipline is in what's missing — makes dense OpenBSD writing feel weightier.

Digital gardens

Wiki-style, bidirectionally linked — but treated as a designed object, not a notes dump.

  • Maggie Appletonmaggieappleton.com

    Hand-drawn illustration and a Seedling / Budding / Evergreen growth badge on every note. Confidence and care become part of the visual grammar rather than buried metadata.

  • Gwern Branwengwern.net

    Custom sidenotes, hover-preview popups for every internal link, drop caps, per-essay confidence/importance ratings. The reader toolbar treats the page itself as configurable furniture.

  • Tom Critchlowtomcritchlow.com

    A wild garden, loosely tended. The structure mimics a literal folder of folders — exposing the filing system as the interface. The deliberate roughness reads as a workshop you're allowed inside.

  • Azlen Elzanotes.azlen.me

    Matuschak-style stacked sliding panels — drill sideways through linked notes without losing the previous one. Treats the screen as a desk of overlapping index cards.

  • Winnie Limwinnielim.org

    Notes, journal, library, poetry, and curated greatest-hits "playlists" sit side by side. Reads as several rooms in one house — a slow, lived-in space where threads accrete over years.

Academic homepages

Researcher pages that escaped the Bootstrap 3 faculty-page fate.

  • Pascal Michaillatpascalmichaillat.org

    Economist who designed his own Hugo template. Tiny horizontal text-link nav, no hero, papers as plain dated bibliographic entries. The unified visual system across his site, slides, CV, and papers is the real flex.

  • Ana Dodikanadodik.github.io

    MIT graphics PhD. Restrained grayscale palette, card-based publications with rendered thumbnails from her actual research, consistent venue/year typography, uniform action chips (paper / video / bibtex).

  • Kelvin Guukelvinguu.com

    Defiantly plain — text-only nav, a narrative bio that integrates accomplishments instead of listing them, publications with proper author asterisks and italicised venues. The confidence to leave it sparse.

  • Linus Lee — thesephistthesephist.com

    Narrative-paragraph homepage instead of a CV dump. Talks listed as chronological inline entries. Reads like an essay rather than a profile — which matches the work.

Old-school personal sites

Sites that feel like the web before the web got monetised — still standing.

  • Ross Anderson (Cambridge)cl.cam.ac.uk

    A tilde-URL, no CSS, hyperlinks as navigation, "new" markers next to recent papers, and a footnote boasting 0.07g of CO2 per pageview. A security professor's homepage that argued its politics by being lightweight.

  • E.W. Dijkstra Archive (UT Austin)cs.utexas.edu

    1,300+ scanned handwritten manuscripts indexed by integer. The whole site is a numbered filing cabinet exposed to the public web — navigation feels like browsing someone's actual desk drawers.

  • Robert Harper (CMU)cs.cmu.edu

    No sidebar, no nav menu, no images — just headings (Papers, Talks, Students, Honors) and a "last modified" stamp. Updated into 2026 but visually unchanged since ~2003. Proof you can run a 30-year career on one HTML file.

  • Devine Lu Linvega — XXIIVVwiki.xxiivv.com

    A hand-built personal wiki with its own constructed language, its own runtime (Uxn), and a permacomputing ethos. Webring badges, hand-drawn iconography, no third-party JS. A single person's lifelong Memex.

  • Paul Batchelor — pbat.chpbat.ch

    "Teaches computers how to sing." Plain text, hyperlinks, a wiki, a webring badge. Fits in a glance and tells you exactly who the person is without a hero image or call-to-action.

Photographers & visual artists

Where the site design itself stays quiet so the work breathes.

  • Alec Sothalecsoth.com

    A text-first index page that treats the portfolio almost like a printed CV: project links in a single left-aligned column with press quotes flowing beneath. The work opens from a quiet table of contents rather than a hero image.

  • Rinko Kawauchirinkokawauchi.com

    Bilingual EN/JA horizontal nav and a single slow-paced image carousel as the entire homepage. The pacing — one frame at a time, generous margins — mirrors the breathing rhythm of her photobooks.

  • Max Siedentopfmaxsiedentopf.com

    Artist's name as both wordmark and section header, with a single category filter row above a uniform grid. The repeated typographic anchor creates rhythm across an enormous body of work without ever shouting.

  • Hassan Rahimhassanrahim.com

    Not a portfolio — a numbered, vertically stacked text document with → and ↓ glyphs doing all the navigation. Reads like a printed colophon, deliberately rejecting scroll-jacked galleries.

  • Gregor Halendagregorhalenda.com

    Five tight category labels over a single uniform image grid where every tile is the project cover. The decision to refuse a hero — every image is equal weight — is what makes it feel edited rather than dumped.

Indie / found gems

Recent picks from the SiteInspire / Godly / Sidebar.io scene that earned their feature.

  • Rauno Freibergrauno.me

    A single column of plain text links with absurd amounts of breathing room — but each project link reveals a custom-built micro-interaction page (cmdk, Vaul). The craft hides behind the click.

  • Nan Liu — nan.fyinan.fyi

    A card grid that looks plain at thumbnail scale, but every post is an interactive explainer. The decision: let the index be boring on purpose so the article pages can be the surprise.

  • Low-Tech Magazine (solar)solar.lowtechmagazine.com

    Honourable adjacent mention. Dithered images, a visible battery gauge in the header, occasional offline hours. A site whose layout is its argument.
